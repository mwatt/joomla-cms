What's New in Version 4.5.3

This version includes many new features including administrator translation, security hardening, new components, interface improvements and many bug fixes.

Language Management

This is the first version of Joomla! to include a complete translation of form labels and general text for both the site and administrator. The language files have been coverted to a simple 'ini' format although the old 'php' format is still supported (but not recommended).

Configuration Variables

mosConfig_debug_db
Displays debug information if there is an error in a query. Recommended that this be on when developing custom work.
mosConfig_debug_dblog
Displays a log of the queries executed.

Administrator Interface

Important: Whenever you enter most "edit" screens, that is, a screen where there is a "Save" button, the menubar will be hidden from view.  This is a safety measure to ensure that a user does not simply navigate to another screen without either saving or cancelling, leaving the item "checked out".

Templates

Many list screen now include the ability to sort by column headings.

Site Templates

Solar Flare II is a new template provided with this release.

Help System Changes

System Info has been removed from the help page and placed back in the 'System' menu.

Components

Global Check-in

Global Check-in has been revised to list those items that are checked-out and allows you to selectively check-in items. The component would previously check-in any items without confirmation.

Export

An Export component is new in this version. While it can serve as a data backup tool, it also allows for the exporting of data in other formats (for example, XML, XMI, etc).

Language Manager

The Language Manager has been completely revised to facilitate the maintenance of all language files.

Mambots

User Authentication

User authentication has been completely revised in this version to allow for pluggable authentication methods, for example, to an LDAP server (a simple example of which is included).

API

mosAbstractTasker

The mosAbstractTasker class has been improved in the following ways:

  • Public methods are now automatically mapped to tasks. For example, a method called editXML will automatically be mapped to a task of editxml (case-insensitive).
  • Tasks can be mapped to the ACL.

phpGACL

The phpGACL library has been upgrade to the latest version and involves some column name changes as well as some changes to function definitions.

The following column names have changed:

  • Table mos_core_acl_aro: aro_id is now id.
  • Table mos_core_acl_aro_groups: group_id is now id.
  • Table mos_core_acl_aro_sections: section_id is now id.
The mos_core_acl_aro_groups includes a new value field.

The following API function have changed their argument list:

  • get_object_groups( $object_id, $type, $option )
  • get_group_id( $value, $name, $type )

These changes now allow you to use the administration client provided with phpGACL providing the connection settings are set appropriately.

patTemplate

There are three examples of administator components using patTemplate and the mosAbstractTasker class. They are com_language, com_export and com_checkin.
